## Knowledge Check: Events

1. **Events allow one object to alert other objects when something occurs.**

    >**True**. When an event *fires* a *listener* triggers their logic.
    
2. **Any method can be attached to a specific event.**

    >**False**. The listener has to match the event's delegate type and parameter.

3. **An attached method cannot be detached.**

    >**False**. Methods can be attached and detached with the `+=`, and `-=` operators, respectively.
